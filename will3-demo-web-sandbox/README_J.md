＃インクレンダリング

インクジオメトリパイプラインは、ポインター入力データをインクジオメトリに変換します。
サンプルアプリケーションでは、ジオメトリパイプラインが作成され、 `InkCanvas`クラスで設定されます。
`InkCanvasVector`は、ベクター2Dレンダリングのパイプラインを拡張します。
`InkCanvasRaster`はラスター（パーティクル）WebGLレンダリングに特化しています。

パイプラインを使用するには、インクパスのコンテンツを指定するデータレイアウトを作成する必要があります。
以下は、パスポイントがX、Y座標、可変サイズの基本的な「レイアウト」です：

`` `javascript
    var layout = [
        PathPoint.Property.X、
        PathPoint.Property.Y、
        PathPoint.Property.SIZE
    ];
「」

サンプルアプリケーションは、プロパティ（回転、スケール、オフセットなど）を追加します。
* Config.js *は、さまざまな入力デバイスに適したいくつかの異なるサンプル設定を提供します。

## PathPoint Calculator

`calculator`は` InkBuilder`に渡すメソッドです。
ポインターイベントからのデータをパスポイントに変換する方法を定義します。
サンプルアプリケーションは、ポインター入力（ペン、マウス、またはタッチ）に依存するパスポイント計算メソッドのいくつかの例を提供し、インクビルダーに適切なデータを提供します。
それらは* Config.js *スクリプトにあります。

##インクの構築

インクをリアルタイムで作成するには、ポインター入力イベントポインターダウン、ポインタームーブ、ポインターアップを処理し、 `InkBuilder`に入力を提供する必要があります。
* On Begin *フェーズは、pointerTypeに対応するように選択されています。
`drawPath`はすべてのフレームでキャンバスを更新します。
`InkBuilder`から提供される` pathPart`には、追加および予測（または予備）データの2つのコレクションが含まれています。
追加されたデータはストロークの永続的な部分になりますが、予測されたデータは一時的なものであり、現在のフレームにのみ表示されます。

`BrushPalette`はいくつかのブラシを定義します。
`InkBuilder`はレンダリングに使用するブラシを知る必要があります。
ベクターインキングには、ブラシの形状を定義するために使用される `PolygonBrush`が必要です。
ラスターインキングには、 `ParticleBrush`の` spacing`プロパティが必要です。
`spacing`パラメータは、パスの軌道に沿った異なるパーティクル間の距離を指定します。
